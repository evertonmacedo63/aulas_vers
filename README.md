## 📁 Estrutura do Repositório dos Alunos

<div align="center">

🗂️ Organização • 🧩 Atividades • 📝 Roteiros • 📨 Feedback  
<br><br>

<img src="https://img.shields.io/badge/Aluno-Repositório%20Individual-blue?style=for-the-badge">

</div>

Cada aluno terá **seu próprio repositório**, organizado de forma clara para facilitar o aprendizado, a entrega das atividades e o acompanhamento do professor.

A estrutura recomendada é:

```
codigo_da_aula/          → códigos produzidos pelo aluno em cada aula
roteiro_da_atividade/    → roteiros fornecidos pelo professor ou baixados do AVA
feedback_do_professor/   → devolutivas formais em arquivo (quando aplicável)
```

---

## 📂 Detalhamento das Pastas

- **codigo_da_aula/** — onde o aluno coloca tudo o que produziu nas aulas práticas.  
  Exemplo:
  ```
  codigo_da_aula/
      aula01/
      aula02/
      aula03/
  ```

- **roteiro_da_atividade/** — onde o aluno armazena os roteiros das atividades práticas.

- **feedback_do_professor/** — onde você pode deixar devolutivas formais em arquivo.

---

## 📝 Feedback via Issues

Além dos arquivos de feedback, o professor também poderá usar **Issues** para:

- apontar erros específicos  
- sugerir melhorias  
- registrar dúvidas  
- acompanhar revisões  
- dar devolutivas rápidas  

---
### Como funciona na prática

- O professor pode abrir uma *Issue* com título como:  
  **“Feedback – Aula 03 – João Silva”**
- O aluno pode responder, corrigir o código e marcar como resolvido.
- O aluno também pode abrir *Issues* para tirar dúvidas.
